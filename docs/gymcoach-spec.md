# GymCoach - Development specifications

> **Document intended for Claude Code**: build a personal PWA for tracking strength-training workouts with an integrated AI coach.

---

## 1. Overview

### Goal

A personal web application (PWA installable on Android) that lets a single user:
- Track strength-training sessions in real time
- Record loads, reps and RIR per set
- Benefit from an automatic timer between sets
- Visualize progress over time through charts
- Receive a weekly debrief and program adjustments generated by Claude (Anthropic API)

### Target user

**A single user** (the author). No multi-account management, no social features. Simple JWT authentication (1 user in the database).

### Critical usage context

The application is used **in a gym**:
- Network sometimes poor in the basement -> **offline-first mandatory**
- Chalky/sweaty hands -> **large buttons (min 64x64px)**
- Between two sets (30-150s) -> **quick reading, one-tap confirmation**
- Phone that locks itself -> **Wake Lock API mandatory during a session**
- Dark locker rooms -> **dark mode by default**

## 4. Database schema (Prisma)

```prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

model User {
  id           String     @id @default(cuid())
  email        String     @unique
  passwordHash String
  createdAt    DateTime   @default(now())
  programs     Program[]
  sessions     Session[]
  exercises    Exercise[]
}

model Exercise {
  id              String          @id @default(cuid())
  userId          String
  user            User            @relation(fields: [userId], references: [id])
  name            String          // e.g. "Flat barbell bench press"
  muscleGroup     MuscleGroup     // enum
  category        ExerciseCategory // compound / isolation
  defaultRestSec  Int             @default(90)
  notes           String?         // technique, mind-muscle cue
  createdAt       DateTime        @default(now())
  programExercises ProgramExercise[]
  sets             Set[]

  @@unique([userId, name])
}

enum MuscleGroup {
  CHEST
  BACK_WIDTH
  BACK_THICKNESS
  SHOULDERS_FRONT
  SHOULDERS_LATERAL
  SHOULDERS_REAR
  BICEPS
  TRICEPS
  FOREARMS
  QUADS
  HAMSTRINGS
  GLUTES
  CALVES
  ABS
  LOWER_BACK
}

enum ExerciseCategory {
  COMPOUND
  ISOLATION
}

model Program {
  id              String           @id @default(cuid())
  userId          String
  user            User             @relation(fields: [userId], references: [id])
  name            String           // e.g. "PPL Hypertrophy 2026"
  description     String?
  phase           String           // e.g. "Hypertrophy", "Strength", "Metabolic stress"
  isActive        Boolean          @default(false)
  startDate       DateTime         @default(now())
  endDate         DateTime?
  createdAt       DateTime         @default(now())
  updatedAt       DateTime         @updatedAt
  workouts        Workout[]
  sessions        Session[]
}

model Workout {
  id              String            @id @default(cuid())
  programId       String
  program         Program           @relation(fields: [programId], references: [id], onDelete: Cascade)
  name            String            // e.g. "Heavy Upper", "Lower", "Full Body"
  dayOfWeek       Int?              // 1=Monday, 2=Tuesday, etc. (null = flexible)
  order           Int               // order within the week
  exercises       ProgramExercise[]
  sessions        Session[]
}

model ProgramExercise {
  id              String        @id @default(cuid())
  workoutId       String
  workout         Workout       @relation(fields: [workoutId], references: [id], onDelete: Cascade)
  exerciseId      String
  exercise        Exercise      @relation(fields: [exerciseId], references: [id])
  order           Int           // order within the session
  targetSets      Int           // e.g. 4
  targetRepsMin   Int           // e.g. 6
  targetRepsMax   Int           // e.g. 10
  targetRIR       Int           // e.g. 2
  restSec         Int           // rest in seconds
  tempo           String?       // e.g. "3-1-1-0"
  notes           String?       // specific instructions
}

model Session {
  id          String    @id @default(cuid())
  userId      String
  user        User      @relation(fields: [userId], references: [id])
  programId   String?
  program     Program?  @relation(fields: [programId], references: [id])
  workoutId   String?
  workout     Workout?  @relation(fields: [workoutId], references: [id])
  startedAt   DateTime  @default(now())
  finishedAt  DateTime?
  notes       String?   // global session notes
  sets        Set[]

  @@index([userId, startedAt])
}

model Set {
  id            String    @id @default(cuid())
  sessionId     String
  session       Session   @relation(fields: [sessionId], references: [id], onDelete: Cascade)
  exerciseId    String
  exercise      Exercise  @relation(fields: [exerciseId], references: [id])
  setNumber     Int       // 1, 2, 3, 4...
  weight        Float     // kg
  reps          Int
  rir           Int?      // 0-5, null if not reported
  notes         String?   // e.g. "wrist pain", "drop set"
  isWarmup      Boolean   @default(false)
  isDropSet     Boolean   @default(false)
  completedAt   DateTime  @default(now())

  @@index([exerciseId, completedAt])
}

model CoachSession {
  id           String    @id @default(cuid())
  userId       String
  weekStart    DateTime  // start of the analyzed week
  weekEnd      DateTime
  prompt       String    @db.Text // prompt sent to Claude
  response     String    @db.Text // Claude's response
  appliedAt    DateTime? // set if the user applied the adjustments
  createdAt    DateTime  @default(now())
}
```

## 7. API specifications

### POST /api/auth/login
```json
Request: { "email": "...", "password": "..." }
Response: 200 { "ok": true } + httpOnly cookie
```

### GET /api/programs
Lists the programs. `?active=true` for the active program.

### POST /api/sessions
Creates a new session.
```json
Request: { "workoutId": "..." }

```

Records a set.
```json
Request: {
  "exerciseId": "...",

  "weight": 75,
  "reps": 10,
  "rir": 1,
  "isDropSet": false,
  "notes": null
```

Triggers the Claude debrief. Returns markdown.
```json
Request: { "weekStart": "2026-04-27" }
Response: { "id": "...", "response": "## Recap...", "createdAt": "..." }
```

## 8. Important technical constraints

### Wake Lock API

```typescript
// lib/wake-lock.ts
let wakeLock: WakeLockSentinel | null = null;

export async function acquireWakeLock() {
  if ('wakeLock' in navigator) {
    try {
      wakeLock = await navigator.wakeLock.request('screen');
    } catch (err) {
      console.warn('Wake Lock failed:', err);
    }
  }
}

export function releaseWakeLock() {
  wakeLock?.release();
  wakeLock = null;
}
```

To be called at the start of a session, to be released at the end.

### Vibration

```typescript
function vibrate(pattern: number | number[]) {
  if ('vibrate' in navigator) navigator.vibrate(pattern);
}
// Set confirmation: vibrate(50)
// Timer end: vibrate([300, 100, 300])
```

## 10. Final notes for Claude Code

- **Use strict TypeScript everywhere** (no `any`)
- **Shadcn/UI components preferred** over custom components when possible
- **Tests**: optional for the MVP, but the critical functions (progression algorithm, sync) deserve unit tests
- **Commits**: one commit per subtask (e.g. "feat(session): add rest timer with vibration")
- **Branches**: one branch per batch (e.g. `feat/batch-5-session-screen`)
- **Ask the user** to validate each batch before moving to the next
- **Document** non-obvious choices in the code
- **Prisma migrations**: create a migration named per batch for traceability
